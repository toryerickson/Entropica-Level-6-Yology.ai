\subsection{Reversibility Principle (Updated Terminology)}

\begin{tcolorbox}[colback=blue!5, colframe=codeblue, title={\textbf{The Reversibility Principle}}]
\textit{``Autonomy is inversely proportional to irreversibility.''} --- Volume I \S5

\textbf{Key Insight:} Gate actions by recoverability, not perceived danger.
\end{tcolorbox}

\begin{table}[h]

\section{Contribution Governance}
%==============================================================================

\subsection{Mapping to Arbiter Layer}

Contribution governance follows the Arbiter Layer (Layer 2) model:

\begin{enumerate}
    \item \textbf{Proposal Submission} $\equiv$ Escalation request
    \item \textbf{Review Process} $\equiv$ d-CAM deliberation
    \item \textbf{Merge Decision} $\equiv$ Arbiter verdict
    \item \textbf{Precedent Setting} $\equiv$ Heuristic update
\end{enumerate}

\subsection{Pull Request Protocol}

\begin{table}[h]
\centering

\begin{tabular}{@{}lll@{}}

\midrule
Draft PR & Level 1 (self-review) & Author validates locally \\
Ready for Review & Level 2 (peer review) & Passes CI tests \\
Requested Changes & Probation (monitored) & Address feedback \\
Approved & Arbiter verdict & Quorum approval \\
Merged & Precedent set & d-CTM logged \\

\subsection{Testing Requirements}

All contributions MUST include:
\begin{itemize}
    \item Unit test definitions (per affected appendix)
    \item Integration test scenarios (if cross-component)
    \item Invariant specifications (if safety-critical)
    \item Formal proof sketches (for P1--P8 changes)
\end{itemize}

\section{Code of Conduct Enforcement}

\subsection{Mapping to Reflex-Heuristic}

The Code of Conduct functions as a \textbf{social Reflex-Heuristic}:
\begin{itemize}
    \item Positive behaviors = Low $\Delta S$ (system stability)
    \item Unacceptable behaviors = High $\Delta S$ (triggers escalation)
\end{itemize}

\section{Operational Procedures}
%==============================================================================

\subsection{Deployment Checklist}

Before deploying EFM capsules:
\begin{enumerate}
    \item Select deployment profile (Appendix I)
    \item Verify spawn parameters within profile limits
    \item Configure Gardener notification thresholds
    \item Execute \texttt{efm-cli profile validate}
    \item Initialize Genesis ceremony (Appendix J \S3)
\end{enumerate}

\subsection{Emergency Procedures}

\begin{tcolorbox}[colback=red!5, colframe=red!70, title={\textbf{Catastrophic Event Response}}]
See Operator's Guide \S7 for detailed runbooks:
\begin{enumerate}
    \item \textbf{Cascading Failure}: Spawn cascade halt $\rightarrow$ ASG emergency ceiling
    \item \textbf{d-CTM Corruption}: Read-only mode $\rightarrow$ Branch replay
    \item \textbf{ZK-SP Chain Break}: Halt affected trunk $\rightarrow$ Forensic reconstruction
    \item \textbf{Gardener Unavailable}: Dead Hand Protocol $\rightarrow$ Automatic safe mode
\end{enumerate}
\end{tcolorbox}
